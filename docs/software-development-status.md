## Folder Structure Analysis
The project follows a strict separation of concerns using the MVC pattern:

- **`app/`**: Contains the application logic.
    - **`Controllers/`**: Orchestrates data between Models and Views.
    - **`Models/`**: Handles database interactions and business logic.
    - **`Middleware/`**: Security and authentication gates.
- **`core/`**: The "engine" of the application.
    - **`App.php`**: Custom router and front-controller logic.
    - **`Controller.php` / `Model.php` / `View.php`**: Base abstractions.
- **`public/`**: The only publicly accessible directory (Entry point: `index.php`).
- **`views/`**: PHP templates for rendering the UI.
- **`docs/`**: Technical documentation and blueprints.

## Security Review

### SQL Injection Protection
- **Status**: ✅ **Secure**
- **Implementation**: Uses **PDO (PHP Data Objects)** with prepared statements in the base `Model` class. Direct queries are prohibited.

### XSS Protection
- **Status**: 🚧 **Partial**
- **Implementation**: Basic filtering in the router, but needs a robust helper function for escaping output in Views (e.g., `h()` function).

### CSRF Protection
- **Status**: ❌ **Pending**
- **Implementation**: Requires a Middleware layer to generate and validate CSRF tokens for all state-changing requests (POST/PUT/DELETE).

### Authentication Security
- **Status**: ⏳ **Planned**
- **Implementation**: Will use `password_hash()` (bcrypt) and secure session management (ID regeneration).

### File Upload Security
- **Status**: ⏳ **Planned**
- **Implementation**: Will include MIME-type validation, file renaming, and storage outside the web root where applicable.

## Suggested Improvements

1. **Service Layer**: Introduce a Service layer to handle complex business logic, keeping Controllers thin and Models focused only on data.
2. **Environment Variables**: Move database credentials from `config.php` to a `.env` file for better security and environment portability.
3. **Template Engine**: Consider a lightweight template engine or a more robust View helper system for cleaner layouts.
4. **Error Handling**: Implement a global Exception handler to catch and log errors gracefully in production.
5. **Asset Pipeline**: Use a basic asset bundler or versioning system for CSS/JS to prevent browser caching issues during updates.
